# Data Models

Casa de Historias uses three representations of story data depending on the layer:

1. **Story** - Domain model for business logic
2. **StoryFirestore** - Firebase Firestore model
3. **StoryEntity** - Room database entity for local storage

## Domain Model: Story

The main business logic model used throughout the app.

```kotlin
data class Story(
    val id: String,
    val titleEs: String,
    val titleNahuatl: String,

    val narratorName: String,
    val community: String,
    val latitude: Double? = null,
    val longitude: Double? = null,
    val authorId: String = "",
    val tags: List<String> = emptyList(),

    val contentNahuatl: String,

```

See `data/local/entities/StoryEntity.kt:7` for the full definition.

### Room-Specific Fields

<ParamField path="@PrimaryKey id" type="String">
  Room primary key annotation
</ParamField>

<ParamField path="audioPath" type="String?">
  **Local** file path to cached audio (not URL)
</ParamField>

<ParamField path="imagePath" type="String?">

<ParamField path="isFavorite" type="Boolean" default="false">

<Note>
The Room entity stores **local paths** instead of URLs for offline access, and includes a `isFavorite` field not present in other models.
</Note>

## Model Conversions

The repository layer converts between these models:

### StoryFirestore → Story

```kotlin
private fun StoryFirestore.toDomainModel(): Story {
    return Story(
        id = this.id,
        titleEs = this.titleEs,
        titleNahuatl = this.titleNahuatl,
        contentEs = this.contentEs,
        contentNahuatl = this.contentNahuatl,
        audioUrl = this.audioUrl,
        imageUrl = this.imageUrl,
        narratorName = this.narratorName,
        community = this.community,
        latitude = this.latitude,
        longitude = this.longitude,
        authorId = this.authorId,
        tags = this.tags,
        createdAt = this.createdAt.seconds * 1000 // Convert to milliseconds
    )
}
```

See `data/repository/StoryRepository.kt:88` for Firestore-to-domain conversion.
