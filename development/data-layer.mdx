# Data Layer

The data layer implements the **Repository Pattern** to provide a clean API for data access, coordinating between Firebase (remote) and Room (local) data sources.

## Architecture

```
UI Layer (ViewModels)
        ↓
Repository Layer (StoryRepository)
        ↓
   ┌────────┴────────┐
   │                  │
FirestoreRepository  Room DAO
   │                  │
Firebase Cloud    Local SQLite
```

## FirestoreRepository

Handles all Cloud Firestore operations.

```kotlin
@Singleton
class FirestoreRepository @Inject constructor(
    private val firestore: FirebaseFirestore
) {
    fun getAllStoriesFlow(): Flow<List<StoryFirestore>> = callbackFlow {
        val listener = firestore.collection("stories")
            .whereEqualTo("published", true)
            .orderBy("createdAt", Query.Direction.DESCENDING)
            .addSnapshotListener { snapshot, error ->
                if (error != null) {
                    close(error)
                    return@addSnapshotListener
                }
                val stories = snapshot?.documents?.mapNotNull { doc ->
                    doc.toObject(StoryFirestore::class.java)
                } ?: emptyList()
                trySend(stories)
            }
        awaitClose { listener.remove() }
    }
}
```

See `data/firebase/FirestoreRepository.kt:15` for the full implementation.

#### Search Stories

Client-side search (Firestore doesn't support full-text search):

```kotlin
suspend fun searchStories(query: String): List<StoryFirestore> {
    val snapshot = firestore.collection("stories")
        .whereEqualTo("published", true)
        .get()
        .await()

    val allStories = snapshot.documents.mapNotNull { doc ->
        doc.toObject(StoryFirestore::class.java)
    }

    val lowerQuery = query.lowercase()
    return allStories.filter { story ->
        story.titleEs.lowercase().contains(lowerQuery) ||
        story.titleNahuatl.lowercase().contains(lowerQuery) ||
        story.narratorName.lowercase().contains(lowerQuery) ||
        story.community.lowercase().contains(lowerQuery) ||
        story.tags.any { it.lowercase().contains(lowerQuery) }
    }
}
```

## Error Handling

All repository methods use try-catch blocks:

```kotlin
suspend fun getStoryById(id: String): StoryFirestore? {
    return try {
        val document = firestore.collection("stories").document(id).get().await()
        document.toObject(StoryFirestore::class.java)
    } catch (e: Exception) {
        null
    }
}
```

Methods return:
- `null` for single-item failures
- `emptyList()` for collection failures
- `false` for boolean operations that fail

## Data Flow Example

1. **UI** calls `viewModel.loadStories()`
2. **ViewModel** calls `storyRepository.getAllStoriesFlow()`
3. **StoryRepository** calls `firestoreRepository.getAllStoriesFlow()`
4. **FirestoreRepository** queries Firestore and returns `Flow<List<StoryFirestore>>`
5. **StoryRepository** maps to `Flow<List<Story>>` (domain model)
6. **ViewModel** exposes as `StateFlow<List<Story>>`
7. **UI** collects and renders

This ensures:
- Clean separation of concerns
- Easy testing (mock repositories)
- Type safety across layers
- Single source of truth
